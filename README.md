# 🚖 Ride-Sharing App (OOP)

## Overview
This project is an Object-Oriented Programming (OOP) prototype of a Ride-Sharing Application developed as a university assignment. It models the interactions between riders, drivers, and trips while applying the principles of encapsulation and abstraction.

## Features
- Rider and Driver management
- Trip creation and management
- Encapsulation of sensitive information
- Abstraction through well-defined classes
- Support for driver ratings
- Multiple ride options (e.g., Economy, Premium)

## OOP Concepts Used
- Classes & Objects
- Encapsulation
- Abstraction
- Inheritance (extendable)

## Main Classes
- User(base class)
- Rider
- Driver
- Trip

## Design Highlights
- Sensitive data such as payment information and user locations are kept private.
- Public methods provide secure access to required information.
- The system is designed to support additional ride types, driver ratings, and payment methods without major modifications.

## Learning Outcomes
This project demonstrates how Object-Oriented Programming principles can be used to design a secure, maintainable, and scalable ride-sharing system.
